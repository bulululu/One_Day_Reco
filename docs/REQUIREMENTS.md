# OneDayReco 产品需求文档

> 最后更新: 2026-06-29

## 一、产品愿景

基于 MBTI 人格画像 + 大模型 Agent 能力，为每个人推荐"适合今天"的活动，并到点提醒。

### 产品定位（2026-06-29 补充）

- **任何地点、任何时间**，打开 App 就能找到"现在可以做的事"
- 一个人也能玩（打游戏、看纪录片、居家休闲等）
- **减少决策成本**——小到"中午去哪吃饭"，大到"周末去哪旅游"
- 不只是推荐活动，是推荐"具体到可以直接执行"的方案
- 覆盖全场景：室内/室外、独处/社交、免费/付费、5 分钟到 3 天

### 推荐具体化原则（2026-06-29 补充）

推荐必须**具体到可直接执行**，禁止泛泛而谈。每条推荐应包含：

| 活动类型 | 必须包含的具体信息 |
|---|---|
| 电影 | 电影名称、电影院名称、地址、场次时间、片长、评分（豆瓣/猫眼） |
| 餐厅 | 餐厅名称、菜系、地址、人均价格、评分、当前是否需排队 |
| 户外活动 | 具体地点名称、地址、当前天气适宜度、预计耗时、路线 |
| 居家活动 | 具体内容（如纪录片名称 + B站搜索词）、所需时间、启动门槛 |
| 运动 | 具体运动名称、场馆名称/地址、价格、开放时段 |
| 展览/演出 | 展览/演出名称、场馆、地址、票价、展期/场次 |
| 购物/市集 | 市集名称、地址、特色、开放时间 |
| 游戏 | 游戏名称、平台、类型、单人/多人、预计时长 |

- 推荐语中必须包含上述具体信息，不能只说"去看场电影"或"去公园走走"
- 如无法获取实时数据（如具体场次），需给出获取路径："打开猫眼搜 XXX，最近的场次是…"

## 二、核心需求

### 2.1 MBTI 个性化推荐

- 用户选择自己的 MBTI 类型（16 种）
- 填写社交偏好、预算、通勤容忍度、特殊偏好
- Agent 结合 MBTI 深层含义 + 当日上下文（天气、时间、位置）生成推荐
- 推荐"此刻这个用户心理上能接受"的活动，而非"热门"活动
- 示例：INFP 宅，可以接受周末去人少的地方看一场电影，但去人多就不行

### 2.2 推荐语气与交互设计（2026-06-29 重新设计）

#### 语气原则：朋友对话，不是分析报告

- **MBTI 是幕后引擎**，用户不应感受到"被分析"
- 推荐语必须是第二人称对话式："你今晚要不要…" "外面天气不错，出去走走？"
- 禁止出现"INTP 型人格通常…" "作为 INFP 你倾向于…"这类分析性表述
- 语气亲近、自然、有温度，达到"吸引他去做"的效果

#### 不同 MBTI 的语气差异化

核心目标：**真正懂每个用户**，不同人格用不同沟通风格，让用户觉得"这个 App 真的懂我"。

语气由四个 MBTI 维度组合决定：

| 维度 | 语气特征 | 示例 |
|---|---|---|
| F 型（INFP/ENFJ/ISFJ 等） | 温柔、有情感共鸣、关心感受 | "今天是不是有点累？要不找个安静的地方待会儿？" |
| T 型（INTP/ENTJ/ISTJ 等） | 直接、简洁、给理由 | "今晚有场电影不错，评分 8.5，离家 10 分钟。" |
| E 型（ENFP/ESFP 等） | 热情、活泼、有感染力 | "外面天气超好！快出来！发现一个超有意思的市集！" |
| I 型（INFP/INTJ 等） | 平和、不催促、留空间 | "想出门的话，附近有个展厅人挺少的。不想动也没事。" |
| P 型（INTP/ENFP 等） | 灵活、随意、不施压 | "随便看看？附近有几个选择，你挑。" |
| J 型（INTJ/ENTJ 等） | 有计划性、给出明确安排 | "7 点出发，路线已看好，25 分钟到。" |

组合示例：
- INFP → 温柔 + 平和 + 灵活 + 关心感受 → "今天天气不错，要不要出去走走？不想动的话在家也挺好的"
- ENTJ → 直接 + 热情 + 有计划 + 给理由 → "今晚 7 点有场高分电影，已帮你查好路线，20 分钟到，去不去？"
- INTP → 直接 + 平和 + 灵活 + 给理由 → "今晚有个纪录片挺适合你的，B站一搜就能看，随时可以开始"

#### 推荐卡片设计

- 每张卡片包含：**配图 + 一句话推荐语 + 行动按钮**
- 配图从网上抓取（与活动相关的氛围图/实景图）
- 行动按钮点击后**直接跳转**对应 App：
  - 电影 → 美团/猫眼订票
  - 户外活动 → 百度地图路线
  - 纪录片 → B站搜索
  - 购物/市集 → 美团/大众点评
- 不需要沉浸式滑动，也不是每天只推一个
- **用户需要时推送**，到点提醒

#### 互动仔（AI 伙伴角色）

- 加一个可聊天的互动仔角色，作为用户的"活动搭子"
- 互动仔的语气、性格、表情跟随用户 MBTI 类型
- 用户可以随时跟互动仔聊天："今天不想出门" "有什么好玩的" "帮我换一个"
- 互动仔回复时自然带出推荐，不是冷冰冰地推卡片
- 配色、交互风格也跟随 MBTI 个性化：
  - INFP → 柔和暖色调，圆角，慢节奏动画
  - ENTJ → 利落冷色调，直角，干脆的交互反馈
  - ESFP → 明亮活泼色，动感动画
  - INTJ → 极简黑白灰，克制设计

#### 交互方式：推送 + 对话结合

- 到点了推送一张推荐卡片
- 用户可以对话式回复："不想去" → 换一个推荐
- 用户可以主动说："今天想出门" → 推荐户外活动
- 对话本身即反馈，不需要单独的"喜欢/跳过"按钮

### 2.3 活动库

- 覆盖 8 大类活动：文化娱乐、户外自然、居家休闲、社交聚会、运动健身、学习提升、创意手工、城市探索
- 每条活动标注：人群密度、社交强度、能耗、室内外、时长、预算、计划性、MBTI 友好度、可调节因素、情绪效果
- **需补充字段**：适合时段(time_window)、启动门槛(convenience)、最低准备时间(min_prep_time)、安全提示(safety_note)
- 活动数据后期与美团/支付宝等 App 联动（当前用种子数据，已标注 source_platform 字段）

### 2.4 反馈学习

- 通过对话行为自动收集反馈（接受/拒绝/换一个）
- 反馈数据持久化存储
- Agent 记住用户偏好，后续推荐体现个性化

### 2.5 到点提醒

- 用户需要时推送推荐活动卡片
- 屏幕使用时间触发推送（见第三章）

## 三、屏幕使用时间触发推荐（2026-06-29 新增）

### 3.1 核心逻辑

- 接入抖音、小红书、游戏等 App 的屏幕使用时间数据
- **户外户内动态平衡**：如果用户室内屏幕活动（刷抖音、小红书、玩游戏）时间过长，自动弹出推荐户外活动
- 游戏本身也算一种活动（室内活动）
- 目标：避免用户长时间沉迷室内屏幕活动，通过推荐户外活动拉回来

### 3.2 触发机制

- 监测连续使用时间，超过设定阈值时触发推荐
- 触发时优先推荐户外、低人群密度活动
- 推荐内容考虑当前 MBTI 类型 + 时间段 + 天气

### 3.3 数据源

- 抖音使用时间
- 小红书使用时间
- 游戏使用时间
- 后期可扩展其他 App

- 后期：美团/支付宝 API 联动

## 五、设计迭代记录

### v0.5 设计迭代（2026-06-29）

针对用户反馈"卡片太暗、固定内容不随性格变化、MBTI 重复选择"三个问题进行优化：

- **推荐卡片视觉增强**：卡片视觉区高度从 130px 增至 160px，emoji 从 48px 增至 56px，新增双层径向渐变高光、3 个装饰圆圈、背景大 emoji（8% 透明度），消除"乌漆嘛黑"问题
- **MBTI 个性化固定 UI 内容**：新增 `mbtiPersonaMap`，16 种类型各有不同的快捷操作按钮文案、互动仔状态文字、聊天输入框占位符、入职引导各步骤标题与副标题
  - 示例：ENTJ 快捷操作为"推荐活动/安排今晚/换方案/查路线"，状态"待命中"，占位符"直接说…"
  - 示例：INFP 快捷操作为"推荐个活动/不想出门/换个推荐/安静一点的"，状态"在线"，占位符"跟搭子聊聊…"
- **MBTI 选择持久化**：用户完成入职引导后，MBTI 类型 + 偏好数据保存至 `localStorage`（`onedayreco_user`）
- **返回用户跳过 MBTI 选择**：页面加载时检查 `localStorage`，若已有保存的 MBTI 类型，自动跳过步骤 1（MBTI 选择），直接进入步骤 2（活动偏好选择），主题和个性化内容立即生效
  - 返回按钮可取消重新配置，直接进入主应用
  - "重置"按钮清除 `localStorage` 并回到完整入职引导流程

## 七、Phase 1 迁移计划（2026-06-29 启动）

### 目标

将前端从单文件 HTML（~1400 行）迁移至 React Native (Expo) + TypeScript，实现组件化、类型安全、状态管理。

### 技术栈

- React Native (Expo)
- TypeScript
- Zustand（状态管理）
- React Navigation（路由）
- Tamagui 或 NativeWind（样式）
- MMKV + AsyncStorage（本地存储）

### 组件拆分

| 组件 | 职责 |
|---|---|
| OnboardingScreen | 3 步入职引导（MBTI 选择 → 偏好 → 备注） |
| ChatScreen | 主聊天界面（互动仔对话 + 推荐卡片） |
| CompanionHeader | 互动仔头部（头像、状态、主题色） |
| MessageBubble | 聊天气泡（渐变背景、打字效果） |
| RecommendationCard | 推荐卡片（配图 + 推荐语 + 行动按钮） |
| QuickActions | 快捷操作按钮（MBTI 个性化文案） |
| BreathingLoader | 呼吸圆加载器 |
| ProgressDots | 入职引导进度指示 |

### 数据层

- MBTI 主题 Map（16 种）→ `src/data/themes.ts`
- MBTI 人格 Map（16 种）→ `src/data/personas.ts`
- MBTI 表单 Map（16 种）→ `src/data/forms.ts`
- 活动分类视觉 Map → `src/data/categoryVisuals.ts`
- API 封装 → `src/services/api.ts`
